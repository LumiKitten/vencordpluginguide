  • Git Download ↗

Install pnpm

ℹ️
Make sure all three tools are added to your PATH after installing. The installers usually do this automatically — just restart your terminal afterwards.
During the Node.js and Git installers on Windows, make sure you check the option "Add to PATH" (it's checked by default). After installing, open a new PowerShell or Command Prompt window before running any commands. If node -v or git -v still fails, restart your computer.

    Finding plugins

    Not sure where to find plugins? Here are some good starting points.

    Great places to find Vencord userplugins:

    • github.com/topics/vencord-plugin — Browse all repos tagged as Vencord plugins
    • github.com/topics/vencord-userplugins — Repos tagged as userplugins specifically
    • D3SOX/vencord-userplugins — A popular collection of high-quality userplugins
    • Vencord's Discord server — Often has plugin showcases and sharing

    ⚠️ Always review the code before installing anything.

    A valid Vencord userplugin repo should contain either:
    • A single index.ts or index.tsx file at the root (simple plugins)
    • A folder structure with index.ts/index.tsx inside (complex plugins)

    The file should start with definePlugin({...}) from Vencord's API.

    After running the commands

    Restart Discord — your custom plugins will appear in the Vencord plugins tab.

    1. Open Discord and go to User Settings
    2. Find the Vencord section in the sidebar
    3. Click on Plugins
    4. Scroll down or search for your plugin by name
    5. Enable it using the toggle — some plugins may need a Discord restart
    • Cannot find package "xyz" — Run pnpm install again. Vencord may have added new dependencies.
    • TypeScript errors — The plugin may be outdated and incompatible with the current Vencord version. Check the plugin's GitHub issues page.
    • pnpm not found — Restart your terminal after installing pnpm, or try npm install -g pnpm again.
    • Permission denied (Linux/macOS) — Try prefixing with sudo, or fix npm permissions per the Node.js docs.
